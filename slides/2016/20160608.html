Generics

Entry types can (and should!) be parametrized:

List<String> list = new ArrayList<>();
Set<Integer> set = new HashSet<>();
Map<Integer, String> map = new HashMap<>();



Allows compiler to prevent errors:

List<String> list = new ArrayList<>();
list.add(new Integer(42)); // COMPILE ERROR

Looping

The "for each" loop provides a convenient way to iterate over entries:

for (String name : Arrays.asList("bill", "linda")) {
    // ...
}

Note: HashSet and HashMap make no guarantee with respect to iteration order

Classes

A class is a template for manufacturing objects: 

class Car {
    // member declarations (fields, methods)
}

Car ford = new Car();

Classes

  • A class typically consists of data and methods to access and/or manipulate that data.

  • Encapsulation is the act of hiding implementation details.

  • Encapsulation enables changes in the implementation without affecting users of a class.

Objects

An object is an instance of a class: 

class Car {
    // member declarations (fields, methods)
}

Car ford = new Car();

In this example:

  • new creates an instance of Car

  • new calls default no-argument constructor of Car

  • ford is a variable of type Car

  • ford references an object/instance of class Car

Objects

  • Each object has its own set of instance variables ("born" when object is created)

  • Conventions:

    • Mutator methods (e.g. setName(…)) change object state

    • Accessor methods (e.g. getName()) access/expose object state

Constructors

  • The name of a constructor is the same as the name of the class

  • A constructor is invoked when an object is created with the new operator

  • A constructor initializes an object’s instance variables

  • A class can have multiple constructors

  • The compiler picks the constructor that matches the construction arguments

Composition

Re-using code by stating a "has-a" relationship: 

class Address {

    private String street;

    // more member declarations
}

class Person {

    private Address address;

    // more member declarations
}

Inheritance

Re-using code by stating an "is-a" relationship: 

class Vehicle {
    // member declarations
}
class Car extends Vehicle {
    // provide own member declarations
    // inherit accessible members from Vehicle
    // override non-final members of Vehicle
}

In Java:

  • At most one class can be declared after the extends keyword (no multiple inheritance)

  • All classes implicitly extend Object

Encapsulation

Hiding implementation details:

class Person {

    private int age;

    public boolean isOldEnough() {
        return age >= 16;
    }

}

Allows internal changes without affecting usages:

public boolean isOldEnough() {
    return age >= 18;
}

Enum Types

The enum keyword allows to define a type with a fixed number of possible values:

enum Season { WINTER, SPRING, SUMMER, FALL }

Allows compiler to prevent errors:

void isChristmasSeason(Season season) {
    return Season.WINTER == season;
}

isChristmasSeason(Season.SUMMER); // false
isChristmasSeason(Season.WINTER); // true
isChristmasSeason("Whenever");    // COMPILE ERROR

Interfaces

Interfaces specify a contract ("what") without providing an implementation ("how")¹:

interface DataModelReader {
    DataModel read(File file);
}

They allow to write generic, re-usable code that supports polymorphism:

DataModelReader reader = resolveReader(file.getName());
DataModel model = reader.read(file);




¹ As of Java 8, interface can provide default implementations

Interface Implementations

By implementing an interface, classes declare to fulfill a contract. They do so by providing the required method body ("how"):

class LineBasedDataReader implements DataModelReader {
    @Override
    public DataModel read(File file) {
        // line-based read implementation
    }
}

class ColumnBasedDataReader implements DataModelReader {
    @Override
    public DataModel read(File file) {
        // column-based read implementation
    }
}

Note: In Java, classes can implement 0 to n interfaces

Lambda Expressions

Lambda expressions are a special syntax to implement single-method interfaces:

interface CheckPerson {
    boolean test(Person p);
}

CheckPerson fullAgeCheck = person -> person.getAge() >= 18;

CheckPerson pointlessCheck = person -> true;

CheckPerson secretServiceCheck = person ->
                person.getFirstName().equalsIgnoreCase("james") &&
                        person.getLastName().equalsIgnoreCase("bond");

Functional Programming

  • FP is a certain style of building and structuring programs (OOP is another such style)

  • FP uses functions as re-usable building blocks

  • Functions are pieces of code that accept input and produce output

  • Like in math, functions should have no side effects: Given the same input, they produce the same output

  • Java 8 combines FP with OOP

GUI Components

JavaFX provides a re-usable hierarchy of components:

GUI Containers

  • GUI components can be added to containers (e.g. Pane)

  • Many containers are components themselves, allowing for complex/nested hierarchies

  • Different Pane implementations provide different visual arrangement/resizing behavior ("layout management"):

    • VBox

    • HBox

    • GridPane

    • StackPane

    • etc.

GUI Events

GUI components are observable through listeners, allowing to react to user interactions:

Button button = new Button("Please Click");
button.setOnAction(event -> {
    System.out.println("Clicked!");
});

Properties

Bindings allow to express relationships between properties of JavaFX nodes:

Slider slider = new Slider(0, 200, 100);
Circle circle = new Circle(150, 150, slider.getValue());
circle.radiusProperty().bind(slider.valueProperty());

Custom Drawing

  • The Canvas component provides an empty rectangular image

  • It provides a GraphicsContext that can be used to issue drawing commands

  • A Canvas can be added to the scene graph like any other node
