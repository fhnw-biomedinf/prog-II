OOP & FP

Das Problem

  • Duplikationen sind unerwünscht

  • Code wird schnell umfangreich, komplex & unüberblickbar

  • Code verändert sich

z.B. Minecraft Update 1.7: 500'000 geänderte Codezeilen

Die Lösung

Divide & Conquer

  • Klassen & Funktionen als Bausteine

  • Einzelne Bausteine klein & überblickbar

  • Klares Design ermöglicht Composability


Viele Wege führen nach Rom

  • OOP ist eine Variante unter vielen

  • FP wieder im Aufwind

Die gute Nachricht

Ab Java 8 lassen sich OOP und FP kombinieren
